MetaRx™ 客户入驻 SOP MetaRx™ Client Onboarding SOP

诺瞳 AKESO™ · 镜片业务 · B2B · 版本 1.0 · 机密 诺瞳 AKESO™ · LENS BUSINESS · B2B · VERSION 1.0 · CONFIDENTIAL

⚠ 执行前必须完成本流程 — 无 DRI 不得启动 — 战略项目必须有 GTM 负责人 ⚠ This process must be completed before any project execution — No DRI = no project — Strategic projects require GTM Owner

本 SOP 适用于所有客户询盘，无论来源如何。 This SOP applies to all client inquiries regardless of source.
01 · 核心原则Core Principles
01
无商业承诺，不启动研发No R&D Without Commitment
NRE 协议必须签署，首笔款项收到后方可开展任何研发工作。NRE agreement must be signed and first payment received before any R&D work begins.
02
免费工作仅限初步可行性Free Work Is Limited
免费工作 = 仅限初步可行性。无付款不得进行模拟、设计或打样。Free work = preliminary feasibility only. No simulation, no design, no sample without payment.
03
研发带宽是分配资源R&D Bandwidth Is Finite
研发带宽是分配资源，非无限服务。最多 2–3 个客户 + 2–3 个内部项目。新项目 = 某项目暂停。R&D bandwidth is allocated, not unlimited. Max 2–3 client + 2–3 internal projects. New project = something pauses.
04
销售不得直接推进至研发No Bypassing R&D
销售不得绕过直接推进至研发。所有客户均须通过 Gate 0 资格审查，无论来源如何。Sales cannot bypass directly to R&D. All clients pass through Gate 0 — no exceptions regardless of source.
05
NRE 费用不因引荐人变化NRE Costs Are Fixed
NRE 费用不因引荐人而改变。CEO/CTO 引荐的客户遵循相同的商业结构。NRE costs do not change based on who introduced the client. CEO/CTO referrals follow the same commercial structure.
06
每个项目必须有单一 DRIEvery Project Needs a DRI
每个项目始终有且仅有一位直接责任人 (DRI)。DRI 的交接须明确，不得默认发生。Every project has a single Directly Responsible Individual at all times. DRI transitions are explicit, never assumed.

  • Type B — 待优化Needs Optimization

30 天无回应30 Days No Response

60 天无回应60 Days No Response

09 · 现有客户审查Existing Client Audit

SOP 启动时运行一次，此后每年一次。对每位现有客户进行重新评估。 Run once at SOP launch, annually thereafter. Re-evaluate every existing client against current criteria.

每位客户评估项Per-Client Assessment
  • 今日如何分类？A/B/C 类，第几级What type and tier would they be classified as today?
  • 当前商业结构是否符合本 SOP？Is current commercial structure aligned with this SOP?
  • 研发带宽消耗是否合理？Is R&D bandwidth being consumed fairly relative to value?
  • 当初是否是正确的决定？可以如何改进？Was the original engagement decision correct? How could it have been improved?
前提条件Prerequisites
  • 销售 + CTO 联合评估每位客户Sales + CTO jointly assess each client
  • 任何退出沟通前必须进行法律审查Legal review required before any exit communication
  • 仅在法律允许且商业合理时退出Exit only if legally permissible and commercially justified
  • 重组 = 重新谈判条款以符合 SOPRestructure = renegotiate terms to align with current SOP
✓ 保留✓ Keep
客户符合现有分类标准。如商业条款尚未对齐，请对齐本 SOP。按新结构继续合作。Client meets current classification criteria. Align commercial terms to this SOP if not already aligned. Continue under new structure.
⚠ 重组⚠ Restructure
客户有价值但商业结构不对齐。重新谈判 NRE、MOQ、IP 条款或资源分配以符合 SOP。Client has value but commercial structure is misaligned. Renegotiate NRE, MOQ, IP terms, or resource allocation to bring in line with SOP.
✕ 退出✕ Exit
客户不符合现有标准且重组不可行。必须先进行法律审查。仅在法律允许时退出。Client does not meet current criteria and restructure is not viable. Legal review required first. Exit only if legally permissible.
10 · 行为要求Behavioral Requirements

本 SOP 仅在以下行为改变时才有效。制度本身无法自我执行。 This SOP only works if these behaviors change. Systems don't enforce themselves.

01
销售停止对未经资格审查的客户说"是"Sales stops saying yes to unqualified clients
Gate 0 不可绕过。销售在内部评审前不得向未经资格审查的客户承诺。Gate 0 is non-negotiable. Sales stops saying yes to unqualified clients before internal review.
02
研发停止接受无 NRE 协议的未定义工作R&D stops accepting undefined work without NRE
研发停止接受无 NRE 协议且未收首笔付款的未定义工作。R&D stops accepting undefined work without a signed NRE agreement and first payment received.
03
所有引荐（包括领导层）均走关卡 0All referrals — including leadership — go through Gate 0
所有引荐（包括 CEO/CTO）均须经过关卡 0。无例外，无捷径。All introductions — including CEO/CTO — route through Gate 0. No exceptions, no shortcuts.
04
研发带宽视为有限分配资源R&D bandwidth treated as a finite resource
所有人将研发带宽视为有限分配资源。增加项目意味着移除一个。Everyone treats R&D bandwidth as a finite allocated resource. Adding a project means removing one.
